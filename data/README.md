# data/ — the data contract + layout

This folder is governed by the **two data contracts** of ADR-0057, instantiated for RotorVitals' bearing-vibration
domain.

## Layout

| Path | What | Git |
|---|---|---|
| `raw/cwru/` | the real CWRU 12 kHz drive-end `.mat` recordings | **git-ignored** (link-only, never re-hosted; staged via `scripts/fetch-data`) |
| `examples/records.csv` | a tiny standard-format sample that PASSES Contract 1 (clone-verify, no download) | committed |
| `derived/models/*.onnx` | the trained WDCNN + deep-AE, exported for live in-browser inference | committed (small) |
| `derived/rv-cwru-samples.json` | the REAL held-out 3 HP segments the live diagnosis runs on | committed |
| `derived/rv-learned-metrics.json` | held-out accuracy / confusion / per-class recall / SNR curve / AE thresholds | committed |
| `derived/cwru-benchmark.json` | the unsupervised classical envelope/SES benchmark (confusion + accuracy) | committed |
| `derived/<case>/trace.json` | the compact per-case replay artifact (Contract 2) | committed (regenerated by the pipeline) |
| `derived/manifests/` | per-case `<case>.json` (Contract 2) + the flat `index.json` inventory | committed |

The committed `derived/` artifacts ARE the real outputs of the heavy precompute lane — the DEFAULT pipeline
(`python -m rotorlab.pipeline all`, numpy-only) rebuilds the per-case traces + manifests from them, so a clone
replays without torch or a CWRU download. `pipeline all --retrain` regenerates the ONNX/metrics from `raw/cwru/`.

## CONTRACT 1 — ingestion (raw vibration → pipeline) — the *bring-your-own-vibration* gate

Defined in `data-pipeline/rotorlab/io/contract.py`. A vibration-record descriptor is **accepted** iff it satisfies
the schema; **rejected** with a reason otherwise (never silently coerced); plausible-but-suspicious records are
**flagged** (accepted; the flag travels into the manifest). A separate `validate_signal` guards the raw array.

### Record schema (`examples/records.csv`)

| Column | Unit | Range / rule | Notes |
|---|---|---|---|
| `case_id` | — | non-empty | identifier |
| `fs` | Hz | ∈ {12000, 48000} | 48 kHz is decimated ×4 (FIR, zero-phase) → 12 kHz; other rates → **reject** |
| `channel` | — | ∈ {DE, FE, BA} | drive-end is primary |
| `rpm` | r/min | [600, 3600] → ok; outside → **flag** | RUL/Campbell unreliable outside the band, still diagnosable |
| `load_hp` | HP | integer | CWRU 0..3; arbitrary for BYO |
| `fault_type` | — | ∈ {normal, inner, outer, ball} | other → **reject** |
| `fault_size_in` | in | {0.007, 0.014, 0.021} → ok; other → **flag** | optional; declared, not assumed |

**Raw-signal guard (`validate_signal`):** NaN/Inf → reject · length < 2048 (the window) → reject · clipping
(> 1% of samples pinned at a rail) → reject · per-window flatline (std < 1e-9) → flag + safe-normalize by 1.0.

**Internal normalization:** every record is brought to **z-scored 2048-sample windows at 12 kHz** (the WDCNN input)
and a 64-D log-binned magnitude-spectrum summary (the deep-AE input) — the same encoding the browser reproduces.

## CONTRACT 2 — artifact (pipeline → web)

Each pipeline run writes a compact per-case trace (`derived/<case>/trace.json`, schema `rotorvitals.trace/v1`) and
a manifest (`derived/manifests/<case>.json`, schema `rotorvitals.manifest/v2`) recording the category/kind, seed,
engine+version, the shared learned-tier artifacts, the trace byte size, the measured **lane/gate** verdict,
Contract-1 flags, and the case metrics. `frontend/src/lib/contract.types.ts` mirrors these schemas so any drift
fails the web build. `scripts/check_artifacts.py` enforces manifest⇄trace byte-consistency in CI.

## Provenance / license

The learned + classical tiers are trained/evaluated on the **real CWRU 12 kHz drive-end** data (Case Western
Reserve University Bearing Data Center, link-only redistribution — the raw `.mat` are never committed or re-hosted).
Smith & Randall 2015 (MSSP 64-65:100-131) documents the difficulty/leakage caveats. The synthetic + run-to-failure
cases are **clearly labelled synthetic** (XJTU-SY/MFPT/Paderborn real datasets are roadmap — see `docs/cases/`).
